# Exercice individuel d'écoconception

## Objectif

Vous devez analyser puis améliorer l'écoconception de cette application.

Vous devez conserver le fonctionnement principal de
l'application et justifier les modifications réalisées.

## Travail demandé

### 1. Réaliser un audit GreenIT initial

Analysez l'application avant toute modification.

Créez un fichier `audit-initial.md` contenant :

- les résultats d'un audit GreenIT, EcoIndex ou Lighthouse ;
- le poids de la page ;
- le nombre de requêtes réseau ;
- les principaux problèmes identifiés ;
- des captures d'écran ou résultats permettant de justifier vos constats.

### 2. Étudier et compléter le backlog

Consultez le fichier [`backlog.md`](./backlog.md).

Pour chaque user story que vous souhaitez réaliser :

- vérifiez qu'elle répond à un problème observé pendant l'audit ;
- précisez son niveau de priorité ;
- ajoutez un indicateur mesurable si nécessaire.

Vous pouvez proposer une user story supplémentaire.

### 3. Préparer un plan d'action

Créez un fichier `plan.md` contenant :

- les user stories sélectionnées ;
- les modifications prévues ;
- l'ordre de réalisation ;
- les résultats attendus.

### 4. Réaliser les modifications

Implémentez les améliorations sélectionnées dans le projet.

Pour chaque modification, vérifiez que :

- l'application continue de fonctionner ;
- le besoin utilisateur est toujours satisfait ;
- la modification produit un gain mesurable.

### 5. Réaliser un audit final

Créez un fichier `audit-final.md`.

Réalisez le même audit que lors de l'étape initiale, puis comparez les résultats :

| Indicateur | Avant | Après | Évolution |
|---|---:|---:|---:|
| Score GreenIT, EcoIndex ou Lighthouse | | | |
| Poids de la page | | | |
| Nombre de requêtes | | | |
| Temps de chargement | | | |

Expliquez les améliorations obtenues et les problèmes qui restent à traiter.
